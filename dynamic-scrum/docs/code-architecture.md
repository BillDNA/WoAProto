## Files — the V1 seam-split layout (July 2026)

The two old monoliths (engine.js ~1360 lines, index.html ~3300 lines) were cut along their existing seams into ordered classic scripts — **no ES modules, no bundler, no build step** (they'd kill the double-clickable `file://` copy). Code moved by extraction, verbatim; the decision record and rationale live in `../planning/specs/v1-architecture.md`. Load order is data: filename sort for `engine/` and `content/` in node, the hand-ordered `<script>` tag chain in the browser (`game/test.js` asserts the chain matches the on-disk parts, in order, with `ui/boot.js` last).

## Refactor discipline — the golden-diff oracle

Determinism is the free regression net: the same seed schedule → **byte-identical `balance.js` aggregates**. Before moving code around, capture golden baselines (`node game/balance.js 24 normal` AND `24 easy` — easy-AI noise is enumeration-order-sensitive); every refactor commit must reproduce them byte-identically, on top of `test.js` + `smoke.js` green. Anything that legitimately changes the numbers is NOT a refactor — it's a rules/AI-strength change: **bump `RULES_VERSION`** (`engine/01-core.js`, tracking the rule book header) atomically with its test-pin updates, so playtest data stays apples-to-apples per version.

## Core model (the engine — `game/engine/` parts)

- Hexes: pointy-top axial coords, key `'q,r'`. **Board shapes are data** from maps.js: `rows: [[r,qFrom,qTo],...]` OR an explicit `hexes: [[q,r],...]` set (V0, for irregular outlines); the engine builds the hex set, grid labels, and (for point-symmetric outlines) the rot180 centre. Five built-in shapes: classic 24 (the physical board), compact 19, hourglass 21, ridge 20, spear 23. Grand(37)/Wide(29) were REMOVED June 2026 (Bill: slow, empty, and over the 24-hex laser-cutter ceiling; both armies fully deployed control only 22 hexes). `setBoard()` switches a module-level current shape — always set from `st.boardShape`.
- **Custom outlines (V0 map-roster-and-shapes)**: a map may carry `shapeDef: {label, hexes:[[q,r],...]}` inline — `ensureMapShape(map)` registers it as `'@<map id>'` (rebuilt every call so edits stick) and normalizes `map.shape`; the def travels with the map, so LAN joins and resumes work (renderAll re-ensures from `st.match.maps`). validateMaps enforces the **24-hex ceiling** for shapeDef maps. The editor's **Board hexes** tool carves add/remove (ghost neighbours to add, click to remove; removing a hex drops its terrain, both hexes' facing sides, and any HQ on it); Mirror refuses non-symmetric outlines. Labels on holed rows keep their columns (a gap in a row = a gap in the numbering — tested).
- **Roster deletion** (rewritten Feedback Round 4 Pass 2): every map is a file in `content/maps/`, so deleting one in the maps screen POSTs `/api/deletemap` → the server removes the file + regenerates the manifest, and the UI splices it from the live `E.MAPS`. No tombstones, no built-in/custom distinction for deletion (the `custom:true` flag is now just a badge + a shipped-roster-test exemption). A **floor of 5 maps** (`MAP_FLOOR`) still warns before you delete below a playable roster; the old "Restore deleted built-ins" button is gone. Deleting/saving needs the server. **V1: which maps are in the match pool is the ACTIVE map-set** (`content/mapsets/`, up to 5 named sets edited on the maps screen, saved via `/api/savemapsets`) — it replaced the old per-browser `woa-disabled-maps` preference, so the browser, the LAN peer, and every CLI tool agree on one roster. Without the server, set edits last the session.
- `hexLabel(k)` — human grid reference ('C4'): row letter from top + position from left ON THE CURRENT BOARD. All engine log messages use it; the UI draws it faintly on every hex. Off-board keys fall back to raw coords.
- **Terrain is hex-owned and directional.** A side is keyed `sideKey(hex,dir)` = `'q,r>d'`. Forest in hex X: +1 attack when X's occupant attacks OUT across a covered side. Mountain in hex X: +1 defense when X is attacked across a covered side. Both hexes of one border can each own a piece on their side. Map defs list pieces as `{t, edges:[[q,r,d],...]}`. **River** (`'R'`, revised 0.3 / Feedback Round 3+4): a river no longer blocks support — support crosses freely for both sides (`borderBlocked` no longer checks `'R'`). Instead a river **denies deploy-control extension**: `riverBetween(st,a,b)` reads both hexes' sides and `deployTargets` skips any empty neighbour reached only across a river (Round-4 bug: D2 was wrongly deployable). Attacks/moves/Airdrop cross freely; not barrageable. Editor cycles F→M→R→empty per side. **Feedback Round 1**: rivers now come in the same physical lengths as forest/mountain — stock `R3:2, R2:4` (was single-side `R1`), and the editor groups contiguous river sides into 2s/3s like F/M.
- **A terrain piece must live inside ONE hex with contiguous directions** (like the physical pieces; `pieceProblem()` enforces, `buildTerrain` throws, tests cover). This fixed the June 2026 "yellow lines" bug where old map defs straddled hexes. Don't relax it.
- Trenches: `st.trenches[hex]` is an ARRAY of `{dirs:[d,d+1], owner}` — a hex may hold several trenches (Bill's DoubleTrenchNotAllowed ruling, June 2026), but edges may not overlap each other or that hex's own terrain sides (`trenchOrientations` filters). **Trenches are attacker-support denial (V0 July 2026, replaced the old +1 defense)**: attacker support may not cross a trenched border into the battle hex (`borderBlocked` checks BOTH hexes' trenches on that border); no defense bonus, the attack itself always crosses, defender support unaffected, ownership irrelevant to the rules. `owner` stays UI-only (mat bookkeeping). Barrage targets individual trenches: choice `{trenchHex, trenchIdx}`. **Barrage reaches the whole board** (June 2026, Feedback Round 5): ANY trench or forest is a legal target; the old in/adjacent-to-controlled-hexes zone was removed at Bill's request.
- Combat: attacker = atk + allied support adjacent to battle hex (excluding the attacker itself; dropped only if its border into the battle hex carries a trench — rivers no longer block support as of 0.3) + adjacent-own-HQ +1 (same crossing checks) + own forest side + card mod. Defender = def + support (never blocked) + adjacent-own-HQ +1 + own mountain side. Higher wins, tie kills both. Attack step flags (all data-driven from maps.js, documented in `card-cheatsheet.md`): `mod` (± attacker power), `tieSpare` (attacker survives ties), `noAdvance` (attacker never enters the hex, even on a win — Ordered Withdraw has both flags since June 2026; the HQ still falls without entering). HQ has def 0; capturing (tie included) wins the battle.
- Through-HQ: units adjacent to ANY HQ can move/swap/attack to hexes on its far side; terrain checks use the HQ hex as the crossing hex.
- Cards: 16 per deck, each played card is removed from the game; rest of hand discarded and recycled. House rules: any card may instead resolve as a basic attack or basic reposition (`playCard(st,id,mode)`) — but **basic reposition is refused while a basic attack is possible** (Feedback Round 1: `listAttacks(st,p).length>0` throws; UI greys the button; encourages attacking); **same-type swaps are illegal** (Round-3 ruling, enforced in rules 1.0 — swapping two identical units is a hidden skipped turn, so `listRepositions` only offers cross-type swaps); cards flagged **`noOpener:true`** in maps.js never appear in the opening hand (Feedback Round 1: `drawHand` reads the flag instead of hardcoding Airdrop — deck-editor toggle pending Batch B); mid-card the UI can reset to a turn-start snapshot (UI-level JSON snapshot, `APP.snap`).
- Play metrics: `st.seen[p][cardId]` counts hand appearances (incremented in drawHand); `st.playLog` records `{p,id,mode,turn,seen}` per play (playCard). `st.stats` counts behaviour (attacks/swaps/marches/deploys/firstBlood) and `st.lastSwap[p]` remembers each player's last swap pair (AI anti-shuffle) — all self-heal on old saves via `ensureStats`. balanceMap aggregates per card (plays/wins/simple/firstSight/seenSum/noop — noop stays in the data (accumulator + DB) but the printed card table dropped its Skip% column in Round 2, since the no-skip rule made it near-zero everywhere) AND per battle (attacks, swaps, zeroKill, tiebreak, firstBlood conversion, hex-control-vs-winner, deployedShare → the report's Behaviour/Decisiveness lines, added Feedback Round 6 so degenerate AI play shows up as a number). Keep these updated if turn flow changes.
- **No-op turns are visible** (Feedback Round 5): `st.pending.acted` counts actions actually resolved; if a play ends with 0 the journal logs "finds no opening — the order is spent to no effect" and the playLog entry gets `noop:true` (counted into the card aggregates and the DB; no longer a printed report column). Bill's principle: a player should always feel they can act, so dead turns must be loud and measured.
- **At least one step of a card must be played** (Feedback Round 2): `mustPlayStep(st)` is true when nothing has acted, the current step CAN act, and no later step can — `applyStep` throws on a `{skip}` there, so you can't skip every step to burn an order for free. Steps with no legal option are still auto-skipped (`skipImpossible`); a card where NO step can act still legitimately no-ops. The AI naturally avoids the illegal skip (its `enumerateChoices`/`greedyResolve` already wrap `applyStep` in try/catch); the UI hides the **Skip step** button when `mustPlayStep`. Effect (balance.js 40): first lever to actually raise AI attacks (~4.9 → 6.2/battle) by forcing the least-bad attack when skipping-all was the escape. Interacts with the still-open literal-reposition rule ([[reposition-rule-doesnt-raise-attacking]]).
- Victory: HQ capture, attrition when a player cannot draw (**surviving units on the board** — `fieldScore`: inf 1 / cav 2 / art 3; reserves never deployed count for nothing; tie goes to the second player — June 2026 revision, rule book updated; kills as such no longer score, though `st.vp` still tracks them for stats), or **concession** (`concede(st,p)`, winType 'concession' — counts as a normal battle loss). `concedeAdvised(st,p)` is the foregone-conclusion heuristic (ADVISORY, never enforced): truthy only when the field-score gap exceeds ~3 VP per remaining turn (best plausible swing; multi-action cards can beat it, which is why it only advises) AND no unit/deploy can reach the enemy HQ within `cardsRemaining(st,p)` turns (a live Airdrop keeps hope alive). The UI hints the human; maybeAI auto-concedes; `simBattle` deliberately does NOT concede so balance stats stay full-game. Match: pool of maps travels inside `match.maps`; first to 3 battle wins.
- AI: `aiPlanTurn(st, difficulty)` — **one parameterized engine, personalities as data (V0 ai-variety)**. `difficulty` is a preset name or a raw config `{noise, breadth, replySamples, replyWeight, weights:{...}}`: noise = evaluation randomness, breadth = top candidates re-scored by the enemy's sampled best reply (0 = pure greedy), weights overlay `AI_WEIGHTS` (every evalState/threat/penalty term is a named weight now). **V1 search overhaul (rules 1.0 bump):** per-step branching is cut to a ranked `shortlist` (winning attacks first, advances next, swaps last) instead of the old random 80-cap, and trench facings are scored by `trenchFacing` toward live enemy lanes — the human-facing map of all of it is `human-instructions/ai-heuristic-model.md` (its weight/personality tables are regenerated by `node dev/gen-docs.js`). easy/normal/hard are `AI_PRESETS` rows; extra personalities live in **maps.js `"ai"`** (shipped: brawler, turtle, hawk) and auto-appear in the menu/dashboard AI pickers and `balance.js` (`node balance.js matchup 16 brawler turtle` pits any two; `node balance.js 40 brawler` runs the map report with one). The refactor was verified byte-identical for the three presets across seeds/pairings. Greedy per-step search over cloned states, returns `{cardId, mode, choices}` which the UI replays with delays; knows the basic-attack/reposition fallback (burns lowest CARD_KEEP card); `sampledReplyScore` resamples the hidden hand from public deck+hand contents, NEVER reads the real hand — keep that honesty. A full hard battle sims in ~1s, so it's fine live and in the lab; keep new configs in that ballpark. Three anti-degeneracy guards (Rounds 5-6, now the `noopPenalty`/`antiShuffle` weights + the attrition projection — don't zero them in a preset without re-measuring `balance.js` Behaviour lines):
	- Zero-action plans take a **-80 noopPenalty** (> the -12 fallback bias AND the hard AI's 2-sample reply noise; 25 wasn't enough). At hard it's re-applied at full strength after the 0.3/0.7 blend (`cand.pen`), and candidates are compared on **common random numbers** (fresh same-seeded rng per candidate in the hard loop) — otherwise one candidate randomly eats an Airdrop-by-the-HQ sample (-600) another never saw. Dead-turn share on attack cards (the then-Skip% column): 65-75% → 0; hard turn-1 dead turns: 9% → 0.
	- `evalState` projects the **attrition winner if decks ran out now** (fieldScore diff + tie-goes-to-second, weight ramping as `cardsRemaining` shrinks; replaced the old kill-VP term). The side losing the standstill is pushed to force combat — this killed the swap-dance stalemate (zero-kill battles at hard: 44% → 3%, second-player win 78% → 56%).
	- **Anti-shuffle**: re-swapping the same pair p swapped last time costs -10 in greedyResolve (`st.lastSwap`).

## UI invariants

- Multiplayer = whole-state JSON push/poll; anything you add to state must stay JSON-serializable and self-contained (e.g. match.maps carries full map defs so the joiner needs nothing local).
- Local saves are versioned (`SAVE_V` in `ui/app.js`, currently 3). Bump it whenever old saved states would break (board removed, trench arrays, state shape changed) — resume then silently clears instead of crashing.
- Modes: 'ai' | 'hotseat' | 'net' | 'watch' (AI plays both sides; `maybeAI` re-arms itself at turn end). Rematch button on battle-over restarts a single-map match in the current mode (local modes only).
- Player mats render one slot per physical piece (solid = reserve, dashed = fielded, ✕ = lost — needs trench `owner`) plus a 16-chip spent-orders track per side. Bill's explicit ask: enemy card attrition must be visible at a glance.
- Board FX layer (`capturePre`/`playFX` + slide/pop/ghost/ring/`fxStrike` helpers) is pure flourish wired through `act()` and the AI driver — it must never touch rules and must survive a full board re-render (it animates AFTER renderAll using from/to info captured before applyStep). V0 combat-clarity: attacks draw a **strike arrow** (bends through the HQ on via-attacks) plus rings on every supporter that actually counted (capturePre reads `E.supportFor(...).hexes` — engine truth incl. trench/river blocking; gold = attacker's, steel = defender's).
- Combat-clarity QoL (V0): **hover attack preview** — hovering your unit (idle choose-card, or an attack step's from-highlights) overlays `A vs D` pills on every hittable hex via `showAttackHints`/`attackPreviewsFor`, using `E.computeAttack` so the numbers can never disagree with resolution; hover-only by design (Bill: might be clutter). **One-action trenches** — after picking the hex, each legal orientation is a brass corner knob (hover previews its two edges, one click digs); the old two-edge flow and `ui.trenchFirst` are gone.
- Maps (Feedback Round 4 Pass 2, map-sets V1): the roster IS `E.MAPS` (content files). Saving a map in the editor updates `E.MAPS` in place (`rosterReplace`) AND POSTs `/api/savemap`; deleting POSTs `/api/deletemap` + `rosterRemove`. **The match pool is the ACTIVE map-set** (`E.mapPool()` / `E.MAPSETS`, edited in the maps screen's Map-Sets panel — up to 5 named sets, one active — saved as `content/mapsets/*.js` via `/api/savemapsets`). Export downloads the whole roster as `maps-bundle.json`; Import writes each map back through `/api/savemap`. Editing/deleting/set-saving needs the local server (file writes); double-clicked file:// can play and edit sets for the session but not persist them.
