## Recommendations for these skills

- **run-ticket** (routed via `alignment-pass refine:run-ticket`): a ticket whose critical path includes
  a command longer than ~10 min cannot be completed by a dispatched agent — the agent can neither hold a
  foreground timeout that long nor survive ending its turn. The skill should name the pattern: the
  **runner owns long-running commands** (session-background them, wait across turns), and either
  sandwiches them between two dispatches or completes the mechanical remainder inline under the full
  verify gate. This run did the latter ad hoc; it should be sanctioned prose, not improvisation.
- **run-sprint**: resume-after-interrupt worked purely off board + tree + transcripts; worth one line in
  Step 0 ("a resumed run re-derives state from the board, Finished notes, and the interrupted agent's
  transcript — expect in-flight ticket work uncommitted in the tree").
