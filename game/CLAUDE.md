You are working on a digital version of Bill's physical board game **War of Attrition**.
Read this first; it will save you from re-deriving (or breaking) decisions that are already settled.

## Source of truth

- `../design-docs/War Of Attrition rule book.md` — the rules. When code and rule book disagree, ask Bill; don't silently pick one.

## Files

- `maps.js` — **ALL tunable game data as hand-editable JSON** (`WOA_BUILTIN`): board shapes, map roster, unit stats/counts, trench count, terrain stock, and the full card deck. Loaded by the engine in both browser (script tag BEFORE engine.js) and node (`require`). Bill edits this directly for rapid prototyping — keep it pure JSON with the explanatory header comment. Card ids `airdrop` and the `starting:true` card are meaningful to the engine.
- `engine.js` — ALL rules, state, the three AIs, and the battle simulator (`simBattle`, `balanceMap`). Balance aggregation is factored as `balanceNew`/`balanceAdd` (+ `balanceSeed`/`balanceFP` for the schedule) so the CLI and the in-browser dashboard fold battles through the SAME code — smoke asserts they agree exactly; keep it that way. Pure JS, no DOM, runs in browser (window.Engine) and node (module.exports). Every gameplay change goes here, never in the UI.
- `index.html` — the entire UI in one file: menu, board rendering (SVG), player mats, card hand, map gallery, map editor, in-game balance lab (`runBalanceUI`, chunked sims), the **Balance Dashboard** (`#dashScr`, `DASH`/`renderDash` — the full balance.js report in-browser: sortable per-map + card tables, behaviour/decisiveness lines, red/blue AI pickers, chunked runner with Stop), watch mode, glossary, LAN client, board FX. Talks to the engine only through its exported API.
- `balance.js` — the balance lab CLI: `node balance.js [n] [easy|normal|hard] [name-filter]` per-map reports (custom maps included); `node balance.js matchup [n]` = the luck-o-meter (stronger AI vs weaker; win rate = skill premium). This is Bill's main iteration tool — keep it working.
- `server.js` — zero-dependency node LAN server: static files, room create/join/push/poll (whole-state sync, seq-numbered), and `/api/savemaps` which writes `custom-maps.js`.
- `custom-maps.js` — generated; `window.WOA_CUSTOM_MAPS = [...]`. Never hand-edit semantics; keep its include after engine.js in index.html.
- `custom-deck.js` — generated by the **Deck Editor** (menu → Quartermaster's Ledger); `window.WOA_CUSTOM_DECK = [...cards] | null`. Loads BEFORE engine.js: an inline script swaps it (localStorage `woa-custom-deck` wins over the file) into `WOA_BUILTIN.cards` before the engine snapshots the list — so applying a deck = reload, same contract as hand-editing maps.js. Server endpoint `/api/savedeck` mirrors `/api/savemaps`. Validation (`deckProblems`) enforces: 16 total copies, exactly one `starting` card at count 1, known step types/flags/units, unique ids. Browser-only: node tools (test.js, balance.js) always read maps.js — the editor says so on-screen.
- `test.js` — engine test suite. Run with `node test.js`. Keep it green; extend it for every rules change. Controlled tests pin `TESTMAP` (bare classic board) so the random roster can't break them.
- `../dev/smoke.js` — jsdom UI harness (`node dev/smoke.js` from repo root; `npm i --prefix dev jsdom` once). Boots index.html, plays a battle through the real DOM, pokes mats/editor/journal.

## Core model (engine.js)

- Hexes: pointy-top axial coords, key `'q,r'`. **Board shapes are data** from maps.js: `rows: [[r,qFrom,qTo],...]` OR an explicit `hexes: [[q,r],...]` set (V0, for irregular outlines); the engine builds the hex set, grid labels, and (for point-symmetric outlines) the rot180 centre. Five built-in shapes: classic 24 (the physical board), compact 19, hourglass 21, ridge 20, spear 23. Grand(37)/Wide(29) were REMOVED June 2026 (Bill: slow, empty, and over the 24-hex laser-cutter ceiling; both armies fully deployed control only 22 hexes). `setBoard()` switches a module-level current shape — always set from `st.boardShape`.
- **Custom outlines (V0 map-roster-and-shapes)**: a map may carry `shapeDef: {label, hexes:[[q,r],...]}` inline — `ensureMapShape(map)` registers it as `'@<map id>'` (rebuilt every call so edits stick) and normalizes `map.shape`; the def travels with the map, so LAN joins and resumes work (renderAll re-ensures from `st.match.maps`). validateMaps enforces the **24-hex ceiling** for shapeDef maps. The editor's **Board hexes** tool carves add/remove (ghost neighbours to add, click to remove; removing a hex drops its terrain, both hexes' facing sides, and any HQ on it); Mirror refuses non-symmetric outlines. Labels on holed rows keep their columns (a gap in a row = a gap in the numbering — tested).
- **Roster deletion**: built-in maps can be deleted in the maps screen — same tombstone list as customs, ids `'b:<name>'` — with a **floor of 5 maps** (`MAP_FLOOR`) so a first-to-3 match always has fresh boards; a "Restore deleted built-ins" button un-tombstones them.
- `hexLabel(k)` — human grid reference ('C4'): row letter from top + position from left ON THE CURRENT BOARD. All engine log messages use it; the UI draws it faintly on every hex. Off-board keys fall back to raw coords.
- **Terrain is hex-owned and directional.** A side is keyed `sideKey(hex,dir)` = `'q,r>d'`. Forest in hex X: +1 attack when X's occupant attacks OUT across a covered side. Mountain in hex X: +1 defense when X is attacked across a covered side. Both hexes of one border can each own a piece on their side. Map defs list pieces as `{t, edges:[[q,r,d],...]}`. **River** (`'R'`, V0 July 2026): support never crosses a river border, for EITHER side, read from both hexes' sides (`borderBlocked`); attacks/moves/Airdrop cross freely; not barrageable. Editor cycles F→M→R→empty per side. **Feedback Round 1**: rivers now come in the same physical lengths as forest/mountain — stock `R3:2, R2:4` (was single-side `R1`), and the editor groups contiguous river sides into 2s/3s like F/M.
- **A terrain piece must live inside ONE hex with contiguous directions** (like the physical pieces; `pieceProblem()` enforces, `buildTerrain` throws, tests cover). This fixed the June 2026 "yellow lines" bug where old map defs straddled hexes. Don't relax it.
- Trenches: `st.trenches[hex]` is an ARRAY of `{dirs:[d,d+1], owner}` — a hex may hold several trenches (Bill's DoubleTrenchNotAllowed ruling, June 2026), but edges may not overlap each other or that hex's own terrain sides (`trenchOrientations` filters). **Trenches are attacker-support denial (V0 July 2026, replaced the old +1 defense)**: attacker support may not cross a trenched border into the battle hex (`borderBlocked` checks BOTH hexes' trenches on that border); no defense bonus, the attack itself always crosses, defender support unaffected, ownership irrelevant to the rules. `owner` stays UI-only (mat bookkeeping). Barrage targets individual trenches: choice `{trenchHex, trenchIdx}`. **Barrage reaches the whole board** (June 2026, Feedback Round 5): ANY trench or forest is a legal target; the old in/adjacent-to-controlled-hexes zone was removed at Bill's request.
- Combat: attacker = atk + allied support adjacent to battle hex (excluding the attacker itself; dropped if its border into the battle hex carries a trench or river) + adjacent-own-HQ +1 (same crossing checks) + own forest side + card mod. Defender = def + support (dropped only across rivers) + adjacent-own-HQ +1 + own mountain side. Higher wins, tie kills both. Attack step flags (all data-driven from maps.js, documented in `../design-docs/card-cheatsheet.md`): `mod` (± attacker power), `tieSpare` (attacker survives ties), `noAdvance` (attacker never enters the hex, even on a win — Ordered Withdraw has both flags since June 2026; the HQ still falls without entering). HQ has def 0; capturing (tie included) wins the battle.
- Through-HQ: units adjacent to ANY HQ can move/swap/attack to hexes on its far side; terrain checks use the HQ hex as the crossing hex.
- Cards: 16 per deck, each played card is removed from the game; rest of hand discarded and recycled. House rules: any card may instead resolve as a basic attack or basic reposition (`playCard(st,id,mode)`) — but **basic reposition is refused while a basic attack is possible** (Feedback Round 1: `listAttacks(st,p).length>0` throws; UI greys the button; encourages attacking); cards flagged **`noOpener:true`** in maps.js never appear in the opening hand (Feedback Round 1: `drawHand` reads the flag instead of hardcoding Airdrop — deck-editor toggle pending Batch B); mid-card the UI can reset to a turn-start snapshot (UI-level JSON snapshot, `APP.snap`).
- Play metrics: `st.seen[p][cardId]` counts hand appearances (incremented in drawHand); `st.playLog` records `{p,id,mode,turn,seen}` per play (playCard). `st.stats` counts behaviour (attacks/swaps/marches/deploys/firstBlood) and `st.lastSwap[p]` remembers each player's last swap pair (AI anti-shuffle) — all self-heal on old saves via `ensureStats`. balanceMap aggregates per card (plays/wins/simple/firstSight/seenSum/noop → the card report columns) AND per battle (attacks, swaps, zeroKill, tiebreak, firstBlood conversion, hex-control-vs-winner, deployedShare → the lab's Behaviour/Decisiveness lines, added Feedback Round 6 so degenerate AI play shows up as a number). Keep these updated if turn flow changes.
- **No-op turns are visible** (Feedback Round 5): `st.pending.acted` counts actions actually resolved; if a play ends with 0 the journal logs "finds no opening — the order is spent to no effect" and the playLog entry gets `noop:true` (→ Skip% in the card report). Bill's principle: a player should always feel they can act, so dead turns must be loud and measured.
- Victory: HQ capture, attrition when a player cannot draw (**surviving units on the board** — `fieldScore`: inf 1 / cav 2 / art 3; reserves never deployed count for nothing; tie goes to the second player — June 2026 revision, rule book updated; kills as such no longer score, though `st.vp` still tracks them for stats), or **concession** (`concede(st,p)`, winType 'concession' — counts as a normal battle loss). `concedeAdvised(st,p)` is the foregone-conclusion heuristic (ADVISORY, never enforced): truthy only when the field-score gap exceeds ~3 VP per remaining turn (best plausible swing; multi-action cards can beat it, which is why it only advises) AND no unit/deploy can reach the enemy HQ within `cardsRemaining(st,p)` turns (a live Airdrop keeps hope alive). The UI hints the human; maybeAI auto-concedes; `simBattle` deliberately does NOT concede so balance stats stay full-game. Match: pool of maps travels inside `match.maps`; first to 3 battle wins.
- AI: `aiPlanTurn(st, difficulty)` — **one parameterized engine, personalities as data (V0 ai-variety)**. `difficulty` is a preset name or a raw config `{noise, breadth, replySamples, replyWeight, weights:{...}}`: noise = evaluation randomness, breadth = top candidates re-scored by the enemy's sampled best reply (0 = pure greedy), weights overlay `AI_WEIGHTS` (every evalState/threat/penalty term is a named weight now). easy/normal/hard are `AI_PRESETS` rows; extra personalities live in **maps.js `"ai"`** (shipped: brawler, turtle) and auto-appear in the menu/dashboard AI pickers and `balance.js` (`node balance.js matchup 16 brawler turtle` pits any two; `node balance.js 40 brawler` runs the map report with one). The refactor was verified byte-identical for the three presets across seeds/pairings. Greedy per-step search over cloned states, returns `{cardId, mode, choices}` which the UI replays with delays; knows the basic-attack/reposition fallback (burns lowest CARD_KEEP card); `sampledReplyScore` resamples the hidden hand from public deck+hand contents, NEVER reads the real hand — keep that honesty. A full hard battle sims in ~1s, so it's fine live and in the lab; keep new configs in that ballpark. Three anti-degeneracy guards (Rounds 5-6, now the `noopPenalty`/`antiShuffle` weights + the attrition projection — don't zero them in a preset without re-measuring `balance.js` Behaviour lines):
	- Zero-action plans take a **-80 noopPenalty** (> the -12 fallback bias AND the hard AI's 2-sample reply noise; 25 wasn't enough). At hard it's re-applied at full strength after the 0.3/0.7 blend (`cand.pen`), and candidates are compared on **common random numbers** (fresh same-seeded rng per candidate in the hard loop) — otherwise one candidate randomly eats an Airdrop-by-the-HQ sample (-600) another never saw. Skip% on attack cards: 65-75% → 0; hard turn-1 dead turns: 9% → 0.
	- `evalState` projects the **attrition winner if decks ran out now** (fieldScore diff + tie-goes-to-second, weight ramping as `cardsRemaining` shrinks; replaced the old kill-VP term). The side losing the standstill is pushed to force combat — this killed the swap-dance stalemate (zero-kill battles at hard: 44% → 3%, second-player win 78% → 56%).
	- **Anti-shuffle**: re-swapping the same pair p swapped last time costs -10 in greedyResolve (`st.lastSwap`).

## UI invariants

- Multiplayer = whole-state JSON push/poll; anything you add to state must stay JSON-serializable and self-contained (e.g. match.maps carries full map defs so the joiner needs nothing local).
- Local saves are versioned (`SAVE_V` in index.html, currently 3). Bump it whenever old saved states would break (board removed, trench arrays, state shape changed) — resume then silently clears instead of crashing.
- Modes: 'ai' | 'hotseat' | 'net' | 'watch' (AI plays both sides; `maybeAI` re-arms itself at turn end). Rematch button on battle-over restarts a single-map match in the current mode (local modes only).
- Player mats render one slot per physical piece (solid = reserve, dashed = fielded, ✕ = lost — needs trench `owner`) plus a 16-chip spent-orders track per side. Bill's explicit ask: enemy card attrition must be visible at a glance.
- Board FX layer (`capturePre`/`playFX` + slide/pop/ghost/ring/`fxStrike` helpers) is pure flourish wired through `act()` and the AI driver — it must never touch rules and must survive a full board re-render (it animates AFTER renderAll using from/to info captured before applyStep). V0 combat-clarity: attacks draw a **strike arrow** (bends through the HQ on via-attacks) plus rings on every supporter that actually counted (capturePre reads `E.supportFor(...).hexes` — engine truth incl. trench/river blocking; gold = attacker's, steel = defender's).
- Combat-clarity QoL (V0): **hover attack preview** — hovering your unit (idle choose-card, or an attack step's from-highlights) overlays `A vs D` pills on every hittable hex via `showAttackHints`/`attackPreviewsFor`, using `E.computeAttack` so the numbers can never disagree with resolution; hover-only by design (Bill: might be clutter). **One-action trenches** — after picking the hex, each legal orientation is a brass corner knob (hover previews its two edges, one click digs); the old two-edge flow and `ui.trenchFirst` are gone.
- Custom maps: localStorage (`woa-custom-maps`, tombstones in `woa-deleted-maps`) merged with `window.WOA_CUSTOM_MAPS`; over http they auto-sync to the server file. file:// and http origins have SEPARATE localStorage — that's why Export/Import buttons exist. The server refuses to blank a maps file unless `allowEmpty`.
- The editor paints terrain per SIDE (click inside a hex near its border); its shape dropdown builds itself from `E.SHAPES`; Mirror applies the shape's rot180; saving groups same-type corner-sharing sides of one hex into pieces.
- `index.html?autostart=ai` deep-links straight into a battle (screenshots, quick testing).
- Art pipeline: `game/art/<card-id>.jpg|png` looked up by card id (`artImg` with onerror fallback chain, `ART_STATE` cache; no art = clean text-only card). title.png / table.jpg / board.jpg dress menu, body, and board via CSS with graceful absence. Raw AI renders are heavy: run `dev/optimize-art.ps1` (trims alpha margins, resizes, jpgs; originals swept to gitignored `design-docs/art-originals/`).
- Editor terrain grouping splits long same-hex runs into physical 2s/3s (`splitRun`; a full ring = 3+3 — Bill's TwoSetsOfThree report). validateMaps stays strict about stock sizes.
- Game layout (**V0 layout pass, July 2026 — "2A + topbar scoreboard"**, supersedes the Round-5 arrangement; spec: `specs/layout-v2-implementation.md`): `#main` is three full-height columns — `.sidecol#leftcol` holds **BOTH mats** (opponent top, YOUR mat bottom next to the hand; hotseat/watch = red top/blue bottom; ordered via flex `order` in renderTop, `.mat-divider` seam between), `#centercol` (boardwrap + promptbar + hand), `.sidecol#rightcol` is the **Campaign Journal only** — full height, bound-book chrome (sticky `.jhead` brass plate, spine/ruled-margin/page-edge as `.jbody` backgrounds). Rule of hierarchy: board + hand are primary and never yield; rails are reference and degrade first; surplus width goes to the board (`#rightcol` capped at `clamp(300px,26vw,420px)`).
- **Board is bounded by its hex geometry**: renderBoard sets `--board-ar` from the SVG viewBox; `#board{height:100%; width:auto; aspect-ratio:var(--board-ar)}` so tall/diamond maps no longer stretch full-width parchment. Don't reintroduce `width:100%`.
- **Journal groups by turn** (renderLog): a card play opens a `.jturn` (steps nested in `.steps`); the last two turns render open, older ones collapse to the play line with a `+N ›` affordance (`.toggler`, expand state in `APP.ui.expanded` — UI-only, never saved, SAVE_V untouched). Card-play entries keep `.play`, no-effect lines `.noop`; `.tn` turn numbers sit in the ruled gutter. The overlay mirror (`syncJournalOverlay` → `#journalBody`) strips `.jhead` and re-wires expand via a delegated click handler.
- **Topbar scoreboard (4B)**: left cell `#battleTitle` = `Battle N · "map"` + `YOU · SIDE` chip (ai/net only); centre `#scorecard` = red pips | `#tug` VP tug-bar | blue pips. Tug-bar: solid = `fieldScore` now, hatched = ceiling if every reserve deploys (`fieldScore + Σ reserves×vp`), cream seam = projected front, VP numbers ride the bar ends — mats and topbar agree by construction (both read fieldScore).
- **Responsive ladder** (mats NEVER scroll — `#leftcol{overflow:hidden}`): ≤1280px rails slim (210/300); ≤820px tall mats compact (CSS hides `.spentlbl/.spent`); ≤960px wide or ≤580px tall the journal rail hides and a brass-ringed **floating hamburger** (`#fabJournal`, absolutely positioned in `#boardwrap`) opens `#journalOvr`; ≤720px the left rail hides too and `#fabRosters` mirrors `#leftcol` into `#rostersOvr` (`syncRostersOverlay`), board full-bleed, topbar stacks (flex-wrap). The old topbar Journal button is gone — the fab replaced it.
- Concede (Feedback Round 5): `#btnConcede` in the topbar (hidden in watch mode / battle-over) — confirm dialog via `confirmOvr`, works in ai/hotseat/net (pushes state). renderPrompt shows a brass hint when `concedeAdvised` fires for the live player; maybeAI checks it first and concedes outright.
- Responsive menu (Feedback Round 4): the menu shrinks to fit short screens via `max-height` queries (Bill: shrink, don't scroll; `#menu` keeps `overflow-y:auto` only as a last resort), and the plaque title's font is width-clamped (`clamp(14px,4.4vw,29px)`) with `min-width:0`/`overflow:hidden` on the panel so the one-line `white-space:nowrap` title can't blow the panel past 92vw. Verify layout at small sizes with headless Chrome `--window-size` (classic `--headless` floors the viewport at ~500px wide, so true phone widths can't be screenshotted locally — rely on the width-clamp).
